# Guia de Contribuição

## 1. Objetivo

Este documento define o padrão de contribuição para manter consistência e qualidade.

---

## 2. Padrão de Commits

O repositório segue o padrão Conventional Commits.

Exemplos:

- feat: nova funcionalidade
- fix: correção
- docs: documentação
- refactor: melhoria estrutural

---

## 3. Fluxo de Contribuição

1. Criar branch específica
2. Implementar alteração
3. Atualizar documentação relevante
4. Atualizar CHANGELOG se necessário
5. Abrir Pull Request

---

## 4. Critérios de Aceitação

- Clareza estrutural
- Documentação atualizada
- Coerência com arquitetura existente
- Justificativa técnica quando aplicável
