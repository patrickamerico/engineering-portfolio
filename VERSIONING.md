# 🔢 Versionamento

Este repositório segue o padrão **Semantic Versioning (SemVer)**.

Formato:

MAJOR.MINOR.PATCH

Exemplo:

1.4.2

---

# 📌 Regras de Incremento

## 🔴 MAJOR (1.x.x → 2.0.0)

Incrementado quando há:

- Mudanças incompatíveis
- Reestruturação significativa
- Alteração arquitetural relevante
- Remoção de funcionalidades importantes

---

## 🟡 MINOR (1.1.x → 1.2.0)

Incrementado quando há:

- Nova funcionalidade
- Novo projeto adicionado ao portfólio
- Expansão relevante de documentação
- Evolução estrutural sem quebrar compatibilidade

---

## 🟢 PATCH (1.0.1 → 1.0.2)

Incrementado quando há:

- Correções
- Ajustes pequenos
- Melhorias pontuais
- Atualizações de documentação sem impacto estrutural

---

# 📦 Versão Inicial

A versão 1.0.0 representa:

- Estrutura consolidada do portfólio
- Projetos organizados
- Documentação base implementada
- Padrão de commits definido

---

# 🚀 Estratégia

- Cada evolução relevante deve atualizar o CHANGELOG.md
- A versão deve ser alterada apenas após mudanças consolidadas
- Commits devem seguir o padrão definido em CONVENTIONAL_COMMITS.md

---

> Versionamento claro demonstra maturidade em engenharia de software.
