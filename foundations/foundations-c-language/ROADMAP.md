# 🗺 ROADMAP — Foundations in C

Este roadmap define a evolução planejada do repositório **foundations-c-language**.

O objetivo é evoluir de fundamentos básicos para conceitos mais próximos de programação de sistemas e estruturas de dados clássicas.

---

# 🎯 Fase 1 — Consolidação (Atual)

Status: ✅ Concluído

- [x] Estruturas condicionais (`if`, `switch`)
- [x] Estruturas de repetição (`for`, `while`)
- [x] Funções
- [x] Arrays
- [x] Matrizes bidimensionais
- [x] Manipulação básica de strings
- [x] Uso de `struct`
- [x] Geração de números aleatórios
- [x] Simulações e jogos em modo texto

Arquivos atuais:
- basic_calculator.c
- unit_converter.c
- task_manager_example.c
- tic_tac_toe.c
- hangman_game.c
- random_number_generator.c
- dice_simulator.c
- simple_registration_system.c

---

# 🟡 Fase 2 — Modularização e Organização

Objetivo: aproximar o projeto de uma estrutura mais profissional.

- [ ] Separar código em múltiplos arquivos `.c`
- [ ] Criar arquivos `.h` para declarações
- [ ] Introduzir organização por módulos
- [ ] Melhorar encapsulamento de funções
- [ ] Criar utilitários reutilizáveis

Resultado esperado:
Estrutura semelhante a projetos reais em C.

---

# 🟠 Fase 3 — Ponteiros e Memória Dinâmica

Objetivo: aprofundar no modelo de memória da linguagem C.

- [ ] Introduzir ponteiros
- [ ] Trabalhar com passagem por referência
- [ ] Implementar `malloc`, `calloc`, `realloc`
- [ ] Implementar `free`
- [ ] Evitar memory leaks
- [ ] Simular estruturas dinâmicas

Projetos candidatos:
- Task manager com alocação dinâmica
- Registro de usuários com memória dinâmica

---

# 🔵 Fase 4 — Estruturas de Dados Clássicas

Objetivo: avançar para conceitos fundamentais de Ciência da Computação.

- [ ] Lista encadeada
- [ ] Pilha (Stack)
- [ ] Fila (Queue)
- [ ] Árvore binária simples
- [ ] Busca linear e binária
- [ ] Algoritmos de ordenação (Bubble, Selection, Quick Sort)

Essa fase consolida a base para:

- Algoritmos
- Sistemas
- Engenharia de software de baixo nível

---

# 🟣 Fase 5 — Persistência e Arquivos

Objetivo: aproximar de aplicações reais.

- [ ] Leitura de arquivos (`fopen`, `fscanf`)
- [ ] Escrita em arquivos (`fprintf`)
- [ ] Sistema simples com persistência
- [ ] Serialização básica de structs

Possível evolução:
Sistema de cadastro persistente em arquivo `.txt` ou `.bin`.

---

# 🔴 Fase 6 — Projeto Estruturado Final

Objetivo: consolidar tudo em um mini-projeto robusto.

Sugestões:

- Mini sistema de gerenciamento com:
  - Módulos separados
  - Persistência
  - Estruturas dinâmicas
  - Organização completa

Ou:

- Mini engine CLI estruturada
- Simulador mais complexo
- Sistema com múltiplas entidades

---

# 🧠 Competências Desenvolvidas ao Final do Roadmap

Ao concluir todas as fases, o domínio incluirá:

- Controle completo de memória
- Organização modular em C
- Estruturas de dados clássicas
- Pensamento algorítmico avançado
- Fundamentos sólidos para:
  - C++
  - Rust
  - Go
  - Desenvolvimento embarcado
  - Sistemas operacionais

---

# 📌 Visão Estratégica

Este roadmap transforma o repositório de:

“Coleção de exercícios básicos”

para

“Base estruturada de engenharia de sistemas em C”.
