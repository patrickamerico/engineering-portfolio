# 🤝 Contributing Guide — Foundations Python Core Concepts

Obrigado por considerar contribuir com este repositório.

Este projeto tem como objetivo consolidar fundamentos da linguagem Python por meio de exemplos práticos e progressivos. As contribuições devem manter o foco educacional, a clareza do código e a simplicidade estrutural.

---

# 🎯 Objetivo das Contribuições

As contribuições são bem-vindas quando:

- Melhoram a legibilidade do código
- Corrigem erros ou comportamentos inesperados
- Tornam a lógica mais clara sem aumentar complexidade desnecessária
- Adicionam exemplos alinhados aos fundamentos da linguagem
- Melhoram a documentação

O foco principal é didática e organização.

---

# 📂 Estrutura do Projeto

Cada arquivo `.py` é independente e executável individualmente.

Evite:

- Criar dependências entre arquivos
- Transformar o projeto em um framework complexo
- Introduzir bibliotecas externas desnecessárias
- Aumentar a complexidade além do nível fundamental

---

# 🧱 Padrões de Código

## 🔹 Convenções de Nome

- Arquivos em `snake_case`
- Funções em `snake_case`
- Classes em `PascalCase`
- Variáveis com nomes claros e descritivos

Exemplo:

- `calculate_total`
- `number_guessing_game`
- `HangmanGame`

---

## 🔹 Organização

- Separar responsabilidades em funções
- Manter a função principal simples
- Evitar código duplicado
- Priorizar clareza sobre otimização prematura

---

## 🔹 Estilo

- Seguir a convenção PEP 8
- Indentação com 4 espaços
- Linhas com comprimento razoável
- Espaçamento consistente

---

# 🧪 Execução e Testes

Antes de enviar uma contribuição:

- Execute o arquivo modificado
- Teste diferentes entradas
- Verifique possíveis erros de execução
- Garanta que o programa funcione em Python 3.8+

Não utilize dependências externas sem justificativa clara.

---

# 🚀 Como Contribuir

1. Faça um fork do repositório.
2. Crie uma branch descritiva.
3. Implemente sua melhoria.
4. Teste completamente.
5. Abra um Pull Request com descrição clara do que foi alterado.

Explique:

- O problema identificado
- A melhoria aplicada
- O motivo da mudança

---

# 🧠 Tipos de Contribuições Aceitas

- Refatoração simples
- Melhoria de organização
- Comentários explicativos
- Pequenas melhorias de estrutura
- Correção de bugs
- Adição de novos exercícios fundamentais

---

# ❌ O que Evitar

- Uso de frameworks externos
- Implementações excessivamente complexas
- Arquiteturas avançadas fora do escopo inicial
- Código difícil de ler
- Otimizações desnecessárias

---

# 📚 Filosofia do Projeto

Este repositório prioriza:

- Clareza sobre complexidade
- Fundamentos sobre abstrações avançadas
- Organização sobre improvisação
- Evolução gradual

O objetivo é fortalecer a base antes de avançar para sistemas maiores.

---

# 📩 Dúvidas ou Sugestões

Abra uma issue descrevendo:

- O comportamento atual
- O comportamento esperado
- Sugestões de melhoria

---

Obrigado por contribuir e fortalecer os fundamentos da engenharia de software. 🚀
